## 🚀 הכנות לבדיקה

### 1. ודא שהמיגרציות רצו
```sql
-- Verify tables exist in Supabase SQL Editor
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
AND table_name IN ('client_sessions', 'client_invitations', 'auth_tokens');
```

### 2. ודא משתני סביבה
```bash
# ב-.env
NEXT_PUBLIC_APP_URL=http://localhost:3000
EMAIL_PROVIDER=console
```

### 3. הפעל את הסרבר
```bash
npm run dev
```

## 🐛 בעיות נפוצות ופתרונות

### בעיה: "לא מחובר" כל הזמן
**פתרון:**
- בדוק שה-cookies מאופשרים בדפדפן
- בדוק ש-`sameSite` הוגדר ל-`lax`
- בדוק שאין CORS issues

### בעיה: אימיילים לא מגיעים
**פתרון:**
- בדוק את הקונסול (במצב development)
- ודא ש-`EMAIL_PROVIDER=console`
- אם `resend`, בדוק API key

### בעיה: טעינה אינסופית
**פתרון:**
- פתח את ה-Network tab
- בדוק שגיאות 401/403
- ודא שהטבלאות קיימות ב-DB

### בעיה: "הסשן פג תוקף" מהר מדי
**פתרון:**
- בדוק את `expires_at` בטבלה
- ברירת מחדל: 30 ימים
- שנה ב-`clientAuth.ts` במידת הצורך
